# 言語ポリシー

pylopdf は世界の Python コミュニティに向けたライブラリであり、とりわけ CJK 文書を
重視しています。そのため多言語対応は装飾的な機械翻訳ではなく、製品品質の一部です。

## 重要言語 { #important-languages }

| 優先度 | 言語 | URL | 重要である理由 |
|---|---|---|---|
| P0 | 英語 | `/` | 技術的意味の正本であり、Python エコシステムの共通言語 |
| P1 | 日本語 | `/ja/` | プロジェクトの母語圏であり、CJK 文書の中核ユースケース |
| P1 | 簡体字中国語 | `/zh-cn/` | 活発で大規模な Python 翻訳コミュニティと CJK 文書需要 |
| P1 | 韓国語 | `/ko/` | 活発な Python 翻訳コミュニティと CJK 文書需要 |

英語を意味上の正本とします。一方で、4 言語すべてを同じナビゲーション、ページ構成、
見出し ID、厳格なリンク検査を備えた完全なサイトとしてビルドします。

## 翻訳原則 { #translation-principles }

- 文脈に沿って意味と例を翻訳し、API 名・コード・ファイル名・CLI オプションは翻訳しない。
- バージョン、セキュリティ、ベンチマークの変更は、同じ変更内ですべての P1 言語へ反映する。
- 言語切替後もリンクが維持されるよう、共通の見出し ID を保つ。
- 浅い翻訳を多数増やすより、保守できる完全な翻訳を優先する。
- 利用実績、継続的な要望、保守チームのいずれかが整った言語を第一級へ昇格する。

スペイン語、ブラジルポルトガル語、フランス語、繁体字中国語は候補として追跡します。
判断記録と昇格条件の全文は
[`LANGUAGES.md`](https://github.com/yhay81/pylopdf/blob/main/LANGUAGES.md) にあります。
